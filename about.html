Software Quality Assurance Department

Rationale for Appointment

Software quality assurance can be defined as a planned and systematic pattern of all actions necessary to provide adequate confidence that an item or product conforms to the established product requirements. The process involves a set of activities designed to evaluate the process by which products are developed or manufactured.

I have been appointed as the SQA Officer for TravelWare to help the company ensure that the software developed within our company is planned, designed, developed and maintained to an extremely high standard.

My plans for my new appointment are:

  • Make sure the product for a client is correct and matches the requirements
  • The maintainability or modifibility of the product can be carried out with ease
  • The software should be portable and should be transferable to another system
  • The system itself should be designed for easy testing
  • The usability of the software should require litte effort and it should be easy to get to know how to use it
  • The software should be reliable and should have little to no defects within the system
  • The application itself should be efficient and shouldn't take up too much space within the computer system and the code itself should be reusable so that it can be reused within a different context.

Possible Outcomes

  • Better Requirements for Software
  • More Realistic Schedules
  • Increased Quality of Documentation
  • Better Communication between client and developers
